# BeanVault 豆历 — 咖啡采收日历页面

## Problem Statement

咖啡爱好者想知道"现在什么产区正在采收""什么产区的豆子正新鲜到港"，但需要自行查阅分散的采收日历图。BeanVault 已经存储了全球 13 个产区的采收/到港月份数据，却缺少一个可视化的页面供用户日常查阅和获取购买建议。

## Solution

在底部导航栏新增"豆历"标签页，提供一张全球咖啡采收热力图 + "本月喝什么"推荐区域。用户打开即可看到 12 个月的产区状态概览，选中某月后获得该月推荐产区的卡片（含推荐理由、风味标签、冲煮建议）。

## User Stories

1. 作为咖啡爱好者，我想在底部导航栏看到"豆历"入口，以便快速访问采收日历
2. 作为咖啡爱好者，我想看到一张 13 产区 × 12 月的热力图，以便直观了解全年采收节奏
3. 作为咖啡爱好者，我想看到热力图单元格的颜色区分采收主季/副季/到港主季/到港副季，以便区分不同状态
4. 作为咖啡爱好者，我想热力图默认高亮当前月份列，以便一眼看到本月状况
5. 作为咖啡爱好者，我想点击某个月份列来切换选中月份，以便查看该月详情
6. 作为咖啡爱好者，我想热力图下方有图例说明，以便理解颜色含义
7. 作为咖啡爱好者，我想在"X月喝什么"区域看到 3 张推荐产区卡片，以便知道本月适合买什么豆子
8. 作为咖啡爱好者，我想每张推荐卡片包含产区名称、国旗、推荐理由、风味标签、处理法说明、冲煮建议，以便做出购买决策
9. 作为咖啡爱好者，我想推荐卡片按优先级排序（到港主季 > 到港副季 > 采收主季 > 采收副季），以便优先看到最值得买的豆子

## Implementation Decisions

### 1. 新页面 + 路由

- 新增 `BeanCalendarPage` 页面组件，路由 `/calendar`
- 在 `BottomNav` 中新增第三个标签"豆历"，路由到 `/calendar`
- 底部导航从 2 项变为 3 项等宽排列

### 2. 数据源

- 使用 `src/info/coffee_harvest_calendar_12month_recommendations.json` 作为唯一数据源
- 构建时静态 import（数据量小，无需懒加载），后续可改为按需加载
- 两个关键数据结构：
  - `monthly_status_index`：按月索引，列出每种状态下的产区 ID 列表，用于热力图渲染
  - `monthly_recommendations`：每月 3 张推荐卡片，含 `origin_attributes` 关联数据，用于推荐区域

### 3. 热力图组件

- 行 = 13 个产区（按 JSON 中 countries 顺序），列 = 1-12 月
- 每格根据该月该产区的状态着色：
  - 采收主季 → 橙色实心
  - 采收副季 → 橙色斜条纹
  - 到港主季 → 青色实心
  - 到港副季 → 青色斜条纹
  - 无数据 → 灰色/空白
- 当前月份列有高亮边框或背景色区分
- 选中月份列有另一种高亮样式
- 头部显示月份标签（1月-12月），左侧显示产区名 + 国旗
- 移动端适配：单元格最小尺寸约 24×24px，横向可滚动（13 产区 + 12 列 = 约 340px+ 宽度）

### 4. 月份选择器

- 热力图顶部显示月份标签行，可点击切换选中月份
- 默认选中当前月份（`new Date().getMonth() + 1`）
- 也可在热力图列头上点击选择
- 选择月份后，下方"X月喝什么"区域同步更新

### 5. 推荐卡片区域

- 标题："X月 · 本月喝什么？"
- 展示 3 张卡片（来自 `monthly_recommendations` 中对应月份的数据）
- 每张卡片包含：
  - 产区名 + 国旗
  - 推荐理由（如"主季新鲜到港"）
  - 风味标签（3-5 个 tag）
  - 处理法说明
  - 推荐冲煮方式
  - 产区定位描述
- 卡片样式与现有 BeanCard 风格一致，使用项目设计 token

### 6. 图例

- 热力图下方显示 4 种状态的图例
  - 🟧 采收（主季） / 🟧 采收（副季）
  - 🟩 到港（主季） / 🟩 到港（副季）
- 副季用斜条纹或纹理区分

## Testing Decisions

### 测试策略

- 仅测试外部行为：月份切换、热力图渲染、推荐卡片数据匹配
- 组件测试：`BeanCalendarPage` 月份切换逻辑、推荐卡片根据选中月份更新
- 数据完整性：验证 JSON 导入的 `monthly_status_index` 包含 12 个月、`monthly_recommendations` 每月有 3 张卡片
- 快照测试：可选，仅用于热力图颜色映射逻辑

### 测试 seams

- 最高 seam：`BeanCalendarPage` 组件 — 输入月份 → 输出热力图数据 + 推荐卡片数据
- 数据层 seam：将 JSON 数据处理逻辑抽为纯函数，独立测试（月份 → 热力图矩阵、月份 → 推荐卡片列表）

## Out of Scope

- 不含用户个人豆子数据与采收日历的关联（如"你的豆仓中有 3 款埃塞俄比亚豆子正在采收季"）
- 不含推送通知（如"本月巴拿马瑰夏到港，快去看看吧"）
- 不含外部数据更新（JSON 数据与代码一起发布）
- 不含热力图单元格的详细弹窗（点击单元格暂不弹出产区详情）
- 不含产区详情页跳转

## Further Notes

- 原数据来源为图片 `IMG_2463.JPG` 的视觉解析，到港时间为精品豆到中国市场的估算窗口，实际因处理方式和物流有 1-2 个月浮动
- 数据中"采收"指产地采收时间，"到港"指精品豆到达中国市场的大致时间
- 推荐优先级：到港主季 > 到港副季 > 采收主季 > 采收副季（到港意味着当前可买到新鲜批次）
