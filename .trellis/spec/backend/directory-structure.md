# Directory Structure

> How backend code is organized in this project.

---

## Overview

This project does **not** have a traditional backend server. The "backend" is **Supabase** (Backend-as-a-Service): PostgreSQL database + Auth + Row Level Security. All backend-relevant code lives in `src/supabase/` and the `supabase/` directory at the project root.

---

## Directory Layout

```
src/supabase/                     # Client-side Supabase code
├── client.ts                     # Supabase client init + auth functions
└── sync.ts                       # CRUD operations + row<->bean mapping

supabase/                         # Supabase configuration (for CLI)
└── (migrations, config, etc.)    # Generated by supabase CLI
```

---

## Module Organization

- **`src/supabase/client.ts`**: Supabase client initialization (`createClient`), auth functions (`signInWithOtp`, `verifyOtp`, `signOut`, `getSession`, `onAuthStateChange`)
- **`src/supabase/sync.ts`**: Database CRUD operations (`fetchRemoteBeans`, `createRemoteBean`, `updateRemoteBean`, `deleteRemoteBean`, `permanentlyDeleteRemoteBean`), auth helpers (`getCurrentUserId`, `isLoggedIn`), and row conversion helpers (`beanToRow`, `rowToBean`)

The Supabase client is configured with:
- `autoRefreshToken: true` — automatic token refresh
- `persistSession: true` — session stored in browser
- `detectSessionInUrl: true` — handles OTP callback in URL

## Naming Conventions

| What | Convention | Example |
|------|-----------|---------|
| Supabase client file | `client.ts` | `src/supabase/client.ts` |
| Sync operations file | `sync.ts` | `src/supabase/sync.ts` |
| CRUD functions | `<action>Remote<Entity>` | `createRemoteBean`, `fetchRemoteBeans` |
| Auth functions | `<verb><Noun>` | `signInWithOtp`, `verifyOtp`, `signOut` |
| Auth check | `is<Condition>` | `isLoggedIn` |
| Row converters | `<entity>ToRow` / `rowTo<Entity>` | `beanToRow`, `rowToBean` |
| Table name constant | UPPER_CASE | `const TABLE = 'beans'` |

## Examples

- Well-organized client init: `src/supabase/client.ts` — single `createClient` call with auth config, exported as `supabase` singleton
- Clean CRUD pattern: `src/supabase/sync.ts` — each function returns `boolean` for success/failure, logs errors with `[sync]` prefix
